  • Errors are displayed inline, when they leave the field. Never during input.
  • Missing information is displayed when clicking the button. Errors are displayed during input.
  • Errors must explain to the user as to WHAT is wrong, so they are lead to what to do to correct it.
  • Errors positioned close to the field explain what they need to do. Global errors simply catch their attention to that there is something wrong.
  • Don't mark required fields, if it's there the user expects it should be filled out. Instead, mark if it's optional.
  • If less than 5 options, use radio buttons instead of a select.
  • Buttons should never communicate errors. Instead they should communicate what the button will do.
  • Only one primary button, and that should move the user forward.
  • Strive for short and clear texts.
  • Don't use placeholder and label, always either or.
